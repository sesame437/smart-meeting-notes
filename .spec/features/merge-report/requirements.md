# Merge Report — Requirements

## 概述
用户在会议列表页勾选多个已完成会议，点击"合并生成报告"，弹出对话框可输入自定义提示词（如"总结本周所有项目进展"），系统合并所有选中会议的转录文本，调用 Bedrock 生成一份新的汇总报告，存入 DynamoDB，可在列表查看、发送邮件。

## 用户故事
- 作为用户，我希望勾选多个已完成会议并合并生成汇总报告，以便获得跨会议的综合分析。
- 作为用户，我希望输入自定义提示词来引导汇总报告的生成方向，以便满足不同场景需求。
- 作为用户，我希望合并报告与普通会议一样可查看和发送邮件，以便统一管理。

## 功能需求

### FR-1：前端勾选 + Action Bar
- 会议列表页每行新增复选框，仅 `reported` / `completed` 状态的会议可勾选。
- 勾选 >= 2 个时，底部出现浮动 action bar，显示已选数量和"合并生成报告"按钮。
- 取消全部勾选后 action bar 隐藏。

### FR-2：自定义 Prompt 弹窗
- 点击"合并生成报告"后弹出模态对话框。
- 对话框包含：
  - 已选会议列表（标题 + 日期，只读）
  - 自定义提示词 textarea（可选，placeholder 示例："总结本周所有项目进展"）
  - "生成"和"取消"按钮

### FR-3：后端合并生成 API
- `POST /api/meetings/merge`
- Request body: `{ meetingIds: [...], customPrompt: "..." }`
- 后端从 S3 拉取各会议转录文本，合并后注入 customPrompt，调用 Bedrock 生成汇总报告。
- 创建新的 DynamoDB 记录：`meetingType: "merged"`, `parentIds: [...]`。
- 生成完成后走现有 export-worker 发送邮件。

### FR-4：合并报告展示
- 合并报告在列表中标记为 "Merged" 类型（新 badge 颜色）。
- 详情页展示与普通报告一致，额外显示"源会议"列表（parentIds 对应的会议链接）。

## 非功能需求
- 合并转录文本可能很大，需截断处理（复用 `truncateTranscript` 逻辑）。
- 最多支持同时合并 10 个会议（防止 token 超限）。
- customPrompt 最大长度 500 字。

## 边界条件
- 选中的会议中如有无转录文件的，跳过并在响应中提示。
- 所有选中会议均无转录文件时返回 400 错误。
- 合并报告不可被再次合并（前端不显示 merged 类型的复选框）。
