# Tech Debt Finder

扫描当前代码库，找出技术债务，输出优先级清单。不修改任何文件。

## 扫描内容
1. 重复代码：相似逻辑出现 3+ 次的模式
2. 过长函数：超过 50 行的函数（列出文件:行号）
3. 遗留注释：TODO / FIXME / HACK / XXX（列出所有位置）
4. 测试覆盖盲区：有业务逻辑但无测试的模块
5. 潜在安全问题：eval/exec使用、未校验的用户输入、硬编码密钥

## 输出格式
### 高优先级（影响稳定性/安全）
- [ ] 文件:行号 - 问题描述

### 中优先级（影响可维护性）
- [ ] 文件:行号 - 问题描述

### 低优先级（代码整洁）
- [ ] 文件:行号 - 问题描述

最后给出一个总体建议：下一步最该解决哪一条，为什么。
